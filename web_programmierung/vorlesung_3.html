Vorlesung 3 - Thema

Wie auch die letzte Vorlesung begann auch diese mit einem kurzen Teil zu HTML. Dieses mal ging es um Hyperlinks, die sogenannte a-Tags (anchor). Grob zusammengefasst haben wir uns lediglich damit auseinander gesetzt wie Verlinkungen innerhalb Websites funktionieren. 

                            
    <a href="link adresse">Linktext</a>

Nicht behandelt haben wir jedoch weitere zur Verfügung stehende Argumente des a-Tags. Eines dieser Argumente, dass auch im Kontext dieser Seite sehr oft Anwedung findet ist das target-Argument. Dieses bestimmt wie der Link geöffnet werden soll. So ist es beispielsweise gängige Praxis Links zu fremden Inhalten folgendermaßen zu verlinken: 

                            
    <a href="google.de" target="_blank">Google</a>

Hier wird beispielsweise auf Google verlinkt und zu diesem Zweck das Target _blank eingesetzt, welches den Link in einem neuen Tab öffnet.

Der interessantere Teil

Nun ging es aber zum interessanten Teil - JavaScript.
Das Thema heute waren vor allem die Ähnlichkeiten und Unterschiede zu Java. Das ergibt durchaus Sinn, da wir uns in den ersten beiden Semestern iim Rahmen der Module Progrmmierung I und Programmierung II bereits intensiv mit Java auseinandergesetzt haben. Den Anfang davon haben wir schon in den ersten beiden Vorlesungen gesehen. Es geht um Functions. 

                            
    function meineFunktion(meinParameter){
        alert(meinParameter);
    }

Diese umfassen das Schlüsselwort function, den Namen der Funktion und eine in Klammern angegebene Liste an Parametern. Hier muss beispielsweise beim Aufruf der Funktion ein Parameter mitgegeben werden, der anschließend mit Hilfe des alert-Befehls wieder ausgegeben wird. Hier trat auch der erste gravierende Unteschied zu Java auf. Und zwar muss man in JavaScript nicht zwingend jeden mitgegebenen Parameter definieren, sondern mit diesen "dynamisch arbeiten". Was das konkret bedeutet würde ich gerne an einem Beipiel zeigen. 

                            
    function addieren(){
        var summe = 0;
        for (var i = 0; i < addieren.arguments.length; i++){
            summe += addieren.arguments[i];
        }
        alert(summe);
    }

Diese Funktion kann mit beliebig vielen Parametern aufgerufen werden. Die übergebenen Parameter stehen uns anschließend als Array mit dem Namen arguments zur Verfügung über welches anschließend mit einer for-Schleife iteriert werden kann. In Java musste man dafür mehrere Varianten der gleichen Methode anlegen.
Sonst allerdings war für mich zumindest nicht viel Neues dabei, da sich der Rest überwiegend mit Java deckt. Lediglich eine andere Eigenheit von JavaScript war noch für mich neu. Und zwar war, dass die Definition globaler Variablen auch innerhalb von Funktionen erfolgen kann. Das kannte ich bis jetzt aus keiner anderen Sprache (weder Java, noch C# oder Pascal). 

                            
    // globale Variable
    var x = 0;
    
    function meineFunktion(){
        // lokale Variable
        var y = 1;
        // weitere globale Variable
        z = 5;
    }

Hier sieht man ein Beispiel des oben beschriebenen. Augenmerk soll hier auf das Innere der Funktion gelegt werden. Variablen gelten hier als lokal (also nur im Kontext der Funktion gültig) wenn ihnen das var-Schlüsselwort vorangestellt ist. Variablen ohne dieses Schlüsselwort sind globale Variablen.
Der genaue Sinn dieser Tatsache wurde uns jedoch, zumindest für heute, noch nicht erleutert. Das wird Teil der nächsten Vorlesung.

Kurzer Rückgriff auf HTML

Nachdem wir uns nun verschieden JavaScript Funktionen und Eigenheiten des Syntax angeschaut hatten haben wir gegen Ende der Vorlesung noch mein einen Sprung zurück zu HTML gemacht. Dabei ging es um den form-Tag und seine Bestandteile. Der Form Tag bietet, wie der Name schon suggeriert, die Möglichkeit Formulare in Websites umzusetzen und somit auch einen komfortablen Weg unter Anderem Userdaten abzufragen und anschließend mit JavaScript zu verarbeiten.
Nun stellte sich dabei die Frage, wie man im JS Code an die Referenz für das Formular bekommt und da gibt es im Groben und Ganzen drei Wege:

  1. Man setzt im form-Tag ein name-Attribut und ruft es im JS-Code über document.meinFormular oder document.forms["meinFormular"] auf.
  2. Man setzt ein id-Attribut im form-Tag und erreicht es über document.getElementById("meinFormular")
  3. Man erreicht das Formular über das automatisch vom Browser angelegte forms-Array und den Index des Formulars (beispielsweise document.forms[0])

Einen "besten" Weg gibt es unter den dreien allerdings nicht und so kommt es auf die individuellen Anforderungen an. Ich persönlich bin ein Fan der zweiten Variante, da ich für das stylen mit CSS onehin fast jedem "wichtigen" Element eine eigene id oder class zuweise.

Hat man nun eine Referenz auf das Form Element erhalten kann man über den Punkt Operator die "Kinder-Elemente" eben jenes ansprechen. Ein Beispiel sieht man hier: 

                            
    var benutzername = document.meinFormular.eingabe1.value;

Hier wird ein Wert eines Eingabefeldes in eine Variable benutzername gespeichert. Aber was sind überhaupt diese Eingabefelder? Und wie viele davon gibt es überhaupt?
Das war das Thema des letzten Abschnittes dieser Vorlesung. Aus Gründen der Übersichtlichkeit habe ich Elemente welche laut Herrn Noll ohnehin nicht mehr allzu gebräuchlich sind weggelassen (tpye hidden und type reset).

Der input-Tag

Der input-Tag an sich ist schnell erklärt. Es ist schlichtweg ein Tag der Eingabemöglichenkeiten bereitstellt. Welche genau und wie diese angezeigt wird bestimmt das type-Attribut.

Type Text

Der input type text stellt ein normales einzeiliges Textfeld dar. 

                                    
    <input type="text" name="eingabe" size="50" value="Standardtext">
Type Password

Der input type password ist lediglich ein einzeiliges Textfeld, wie type text jedoch erscheinen Eingaben als Sterne (*). Also wie man es von einem Passwortfeld erwartet.

Type Button

Der type button ist der einzige Typ den ich bisher in diesem Journal verwendet habe. Es handelt sich hier um die Buttons welche sich mit der Maus anklicken lassen. So lassen sich Aktionen auslösen, beispielsweise um den Nutzer um Bestätigung von Eingaben zu bitten oder um anderweitige Funktionen auszulösen. Ein beispiel dafür sieht man hier: 

                                    
    <input type="button" value="Klick mich!" onclick="alert('Hat geklappt!')">

Hier wird beim klicken der Eventhandler onclick mit der Funktion alert aufgerufen.

Type Submit

Type submit ist type button sehr ähnlich mit der Unterscheidung, dass submit eine festgelegte Bestimmung hat - und zwar ein Formular abzuschicken.

Type Checkbox

Der type checkbox ist uns auf anderen Websites definitiv schon mal begegnet. Es handelt sich hier um eine Auswahlbox, die der nutzer "anhaken" kann um, beispielsweise im Kontext eines Formulars, bestimmte Dinge aus- oder abzuwählen (wie den nächsten nervigen E-mail Newsletter oder neu seit diesem Jahr - Coockies). Ausshen tun diese wie folgt:

Auswahlbox 
                                    
    <input type="checkbox">Auswahlbox
Type Radio

Type radio bietet dem Nutzer euch eine Möglichkeit Dinge auszuwählen, jedoch innerhalb einer Gruppe dieser inputs immer nur eines zur selben Zeit. So kann man, wenn man allen Elementen der Radio Gruppe das gleiche name-Attribut gibt, diese "zusammenschalten".
Interessanterweiese bedeutet das auch dass sie dann in JavaScript als das Array radioGruppe mit ihrem entsprehchendem Index adressiert werden können.

Auswahl A Auswahl B Auswahl C Auswahl D 
                                    
    <input type="radio" name="radioGruppe">Auswahl A
    <input type="radio" name="radioGruppe">Auswahl B
    <input type="radio" name="radioGruppe">Auswahl C
    <input type="radio" name="radioGruppe">Auswahl D

Der textarea-Tag

Der textarea-Tag lässt sich als mehrere übereinandergestapelte text-Inputs beschreiben. Simpel ausgedrückt ist es lediglich eine mehrzeilige Texteingabe. 

                                    
    <textarea cols="30" rows="5">
        Standardtext
    </textarea>

Der select-Tag

Der select-Tag war auch mir neu, aber sein Output nicht.
Es handelt sich hierbei um eine Combobox, bei der man zwischenverschiedenen Optionen wählen kann. Die Optionen werden dabei mit Hilfe von option-Tags realisiert. 

                            
    <select>
        <option selected="selected">Standardoption</option>
        <option>Option B</option>
        <option>Option C</option>
        <option>Option D</option>
    </select>
