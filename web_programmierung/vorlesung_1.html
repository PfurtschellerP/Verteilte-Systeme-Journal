Motivation zum Schreiben einer Web Anwendung

Wir begannen damit, darüber zu sprechen welche Vor- und Nachteile das Verteilen von Software als Web Anwendung hat. Dazu haben wir vor allem über die Art der Verteilung (Deployment) bei verschiedenen Ansätzen gesprochen.
Als erstes sprachen wir über das klassische Herunterladen und Installieren einer Desktopanwendung. Diese muss für jedes Betriebssystem und jede Systemarchitektur angepasst werden.
Eine Webanwendung hat den Vorteil, dass sie auf jedem System und jeder Architektur (in den meisten Fällen) ohne Anpassunge lauffähig ist so lange es einen modernen Web Browser für das System gibt.

Vorstellung der Architektur von Webanwendungen

Nun, haben wir uns verschiedene Architekturen zum Implementieren von Web Anwendung angeschaut.
Begonnen haben wir mit einem clientseitigen Ansatz. Dabei liegt en executable (z.B. eine Java Datei) auf einem Webserver welche zur Laufzeit heruntergeladen wird und clientseitig ausgeführt wird. Die Nachteile dieser Methode wurden sofort ersichtlich. Das Ausführen von fremdem Code auf dem eigenen Computer ist nicht umbedingt das, was man sich unter sicher vorstellt.

Danach haben wir uns das komplette Gegenteil ansgeschaut - die serverseitig Programmierung. Dabei wird auf dem Webserver direkt PHP und/oder Java Code ausgeführt und dem Enduser lediglich eine an sich statische HTML Site präsentiert. Dies ist, obwohl sicherer, nicht umbedingt eine Verbesserung in Sachen usability.
Deswegen haben wir einen weiteren Ansatz besprochen. Und zwar den, bei dem die beiden zu erst genannten kombiniert werden. Also wird serverseitig beispielsweise PHP Code ausgeführt während im Browser des Client zusätzlich mit JavaScript gearbeitet wird. Dies sieht dann folgenermaßen aus:

Was ist eine Skriptsprache?

Wir brauchen also Javascript - eine Scriptsprache. Aber was ist das überhaupt. Um das zu verstehen haben wir die drei großen "Sprachkategorien" mal aufgelistet:

  • Programmiersprachen (z.B. Java, C#, etc.)
  • Auszeichnungssprachen (z.B. HTML, XML, etc.)
  • Scriptsprachen (z.B. Javascript, Typescript, etc.)

Dabei unterscheiden sich die Vertreter dieser Kategorien im Umfang ihrer Möglichkeiten wie auch der Anwendung in der Praxis. Während Programmiersprachen wie Java mit Compilern kompiliert werden und zu ausführbaren Anwendungen mit (optional) User Interface werden, können Auszeichnungssprachen wie HTML beispielsweise nur Anwendungen modelieren (ohne Logik).
Skriptsprachen widerum zeichnen sich vor allem durch zwei Dinge aus.

  1. Sie werden nicht kompiliert sondern interpretiert. Das heißt während der Laufzeit wird der Code von einem Interpreter durchlaufen und Zeile für Zeile umgesetzt.
  2. Besitzen eine dynamische Typenprüfung. Das bedeutet es muss sich nicht im vorhinein auf einen Datentyp für eine Variable festgelegt werden (mit allen Vor- und Nachteilen)

Praxis

Nun aber genug der langweiligen Theorie und auf zur Praxis. Gegen Ende der Vorlesung habe wir uns auch mal erste Implementierungen von Javascript angeschaut (und natürlich erste Grundlagen in HTML, die ich mir an dieser Stelle spare, da es ja mein Journal ist [für Referenzen siehe diese Website])
Wir haben uns dazu als erstes den script Tag in HTML angeschaut. Dort haben wir einen alert Befehl eingesetzt, der uns bei jedem Laden der Website eine Meldung ausgibt. 

                            
    <script type="text/javascript">
        alert('Hallo Welt');
    </script>

Da das aber noch relativ langweilig ist (und vor allem unkontrolliert abläuft) haben wir daraufhin noch ein HTML Formular gebaut und den alert Befehl über einen button ausführbar gemacht. Den Code dazu sieht man hier runter: 

                            
    <form>
        <input type="button" value="Test" onclick="alert('Hallo Welt');">
    </form>
